# 功能清单与缺口

本文档记录 `claude_clone` 当前功能边界、已知缺口和可拆分的后续工作。状态含义：

- `[x]` 已实现：代码中已有真实交互或后端路径。
- `[~]` 部分实现：已有 UI 或局部数据流，但能力未闭环。
- `[ ]` 缺失：当前仅占位、未接入真实实现，或还没有入口。

## 已实现基线

- `[x]` GPUI 桌面应用启动、顶栏、侧边栏、Dock 多标签会话布局。
- `[x]` 会话创建、打开、关闭当前/左侧/右侧/其他标签、标题重命名、置顶、删除。
- `[x]` 会话搜索，支持标题和消息内容匹配、键盘上下选择、回车打开。
- `[x]` 会话分支，支持从用户消息或 AI 消息截断生成新会话，并保留来源时间线。
- `[x]` 消息复制、删除、撤销删除、用户消息编辑后重新生成、AI 回复重新生成。
- `[x]` Chat 模式接入 `genai`，支持流式输出、reasoning/thinking 捕获、token 统计、停止生成。
- `[x]` Cowork 和 Code 模式有独立展示形态与模拟后端。
- `[x]` 本地图片附件选择、预览、打开、在文件夹中显示。
- `[x]` 文档附件解析，支持常见 Office/PDF/文本格式；OCR 作为 `document-ocr` feature 的可选路径。
- `[x]` 本地语音输入，使用 `cpal` 实时采集麦克风并通过 `sensevoice` Rust crate 加载本地 GGUF 模型转写到输入框；输入设备可在通用设置中选择并持久化。
- `[x]` 图片生成请求路径，支持从模型返回的图片二进制内容渲染为生成图片块。
- `[x]` 项目创建、会话归档到项目、项目树面板、从项目移除会话。
- `[x]` Artifact 面板汇总图片、生成图片、工具文件等产物，并支持类型、时间、会话过滤。
- `[x]` Provider 管理，支持新增/删除/启停供应商、配置 API Key/Base URL、拉取模型列表、模型分组和选择。
- `[x]` MCP 配置编辑、保存、重载、全局/单服务器启停、stdio MCP 工具调用和工具结果展示。
- `[x]` 语言切换，支持 `en` 和 `zh-CN`。
- `[x]` 持久化设置、Provider、会话历史、项目、Dock 布局、存储目录和配置目录。
- `[x]` GitHub release 自动更新检查、下载、安装和重启入口。
- `[x]` 跨平台打包脚本和应用图标资源。

## P0 缺失功能

- `[ ]` 截屏附件闭环。
  - 现状：“Take a screenshot” 菜单项只显示通知，没有调用系统截屏或把截图加入附件。
  - 验收：可截取屏幕或窗口区域，截图作为附件进入当前会话，并可预览/删除/发送。

- `[ ]` 联网搜索真实能力。
  - 现状：设置和菜单中有 `web_search` 开关，但 Chat 请求没有绑定搜索工具或搜索提供方。
  - 验收：启用后模型可使用搜索工具；回复中能展示搜索调用过程、来源摘要和错误状态。

- `[ ]` 记忆系统。
  - 现状：设置里有 Memory 开关，但没有记忆存储、提取、注入上下文或管理界面。
  - 验收：可保存/查看/删除记忆；发送请求时按开关注入相关记忆；隐私和清空行为明确。

- `[ ]` 主题设置闭环。
  - 现状：设置里有 Theme 分区，但仅渲染 stub。
  - 验收：支持浅色/深色/跟随系统或至少一组可选主题；选择会持久化并立即应用。

## P1 待补全功能

- `[~]` Skills 入口。
  - 现状：菜单项只弹出“已打开技能”的提示。
  - 待补：定义技能来源、启停、权限展示，以及与 Chat/MCP 工具调用的关系。

- `[~]` Connectors 入口。
  - 现状：菜单项只弹出“Connectors panel opened”的提示。
  - 待补：连接器配置界面、凭据存储策略、连接状态检查和工具暴露方式。

- `[~]` Help / Upgrade / Apps / Logout。
  - 现状：用户菜单项均为通知占位。
  - 待补：明确本项目是否需要账户体系；如果不需要，应移除或降级为外部链接/关于页。

- `[~]` Cowork / Code 真实后端。
  - 现状：Chat 模式是真实模型流式请求；Cowork 和 Code 仍走模拟响应。
  - 待补：为 Cowork 定义任务状态机和工具执行流；为 Code 定义项目上下文、文件读写权限和代码操作边界。

- `[~]` 图片生成触发方式。
  - 现状：依赖示例菜单和文本识别触发图片生成，缺少明确模式和参数。
  - 待补：增加显式“生成图片”流程，支持尺寸、数量、风格、模型能力检查和失败重试。

- `[~]` MCP 安全边界。
  - 现状：可读取并启动本地 child-process MCP server。
  - 待补：配置敏感信息遮蔽、命令风险提示、首次运行确认、环境变量展示控制、工具调用审计。

- `[~]` Provider 凭据安全。
  - 现状：Provider API Key 明文写入 `providers.json`，代码注释也标明这是示例级实现。
  - 待补：接入系统 Keychain/Secret Service/Credential Manager，或提供明确的明文存储提示和迁移路径。

- `[~]` Artifact 操作。
  - 现状：Artifact 面板可汇总、过滤、定位产物。
  - 待补：支持导出、复制路径/内容、批量清理、按项目过滤、失效本地文件提示。

## P2 改进清单

- `[ ]` 会话导入/导出。
  - 支持 Markdown/JSON 导出、会话备份恢复、跨设备迁移。

- `[ ]` 快捷键总览与命令面板。
  - 已有局部键绑定，但没有统一发现入口。

- `[ ]` Provider 模型能力标注。
  - 标注模型是否支持图片输入、图片输出、reasoning、tool call、MCP、多模态上下文等能力。

- `[ ]` 请求参数设置。
  - 支持 temperature、max tokens、system prompt、reasoning effort 等参数的全局或会话级配置。

- `[ ]` 对话上下文管理。
  - 支持上下文窗口估算、自动摘要、固定系统提示、按消息排除上下文。

- `[ ]` 项目管理增强。
  - 支持项目重命名、删除、排序、未归档会话分组、项目级默认模型/提示词。

- `[ ]` i18n 覆盖检查。
  - 增加测试或脚本确保 `locales/en.yml` 与 `locales/zh-CN.yml` key 完整一致。

- `[ ]` UI 自动化/截图回归。
  - 对设置、Provider、会话、附件、MCP、项目和 Artifact 面板建立基本回归场景。

- `[ ]` 单元测试覆盖扩展。
  - 当前仅覆盖 `panel_data` 的项目树和 Artifact 过滤；还应覆盖存储、模型排序、搜索匹配、MCP 配置解析、文档路径识别等纯逻辑。

## 建议拆分顺序

1. P0 先补“占位 UI 已暴露但不可用”的能力：截屏、联网搜索、记忆、主题。
2. P1 再补和外部系统/安全相关的能力：Skills、Connectors、MCP 安全、凭据安全。
3. P2 作为产品化体验和测试质量提升，按发布节奏逐步拆分。
